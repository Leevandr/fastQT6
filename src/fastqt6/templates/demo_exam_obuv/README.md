Главный документ для подготовки и самопроверки: `docs/DEMO_EXAM_OBUV_GUIDE.md`.

Быстрый запуск:

```powershell

```

Этот репозиторий создан как чистый старт для подготовки к демоэкзамену. Цель: не повторять старый проект по памяти, а закрывать требования задания как чек-лист, чтобы было понятно, какие баллы уже взяты и что осталось.

Главный принцип: сначала делаем правильную структуру данных и роли, потом интерфейс и CRUD. Если база данных не совпадает с ТЗ, остальные модули начинают терять баллы цепочкой.

## 1. Ориентир по баллам

| Уровень | Примерный порог | Баллы из 75 |
|---|---:|---:|
| 3 | 50% | 38+ |
| 4 | 70% | 53+ |
| 5 | 85% | 64+ |

Твоя цель минимум для уверенной "4": закрыть базу, роли, список товаров, поиск/фильтр/сортировку, CRUD товара и базовые документы. Для "5" нужно почти без пропусков закрыть заказы, изображения, стиль и все подтверждающие файлы.

## 5. Роли и права

| Роль | Что видит | Что может |
|---|---|---|
| Гость | Каталог товаров | Только просмотр товаров, без поиска, фильтрации и сортировки |
| Клиент | Каталог товаров | Только просмотр товаров, без поиска, фильтрации и сортировки |
| Менеджер | Каталог и заказы | Просмотр товаров с поиском, фильтром, сортировкой; просмотр заказов |
| Администратор | Каталог и заказы | Все действия менеджера плюс добавление, редактирование, удаление товаров и заказов |

Проверка для себя: если войти клиентом, кнопки "Добавить", "Редактировать", "Удалить", "Заказы" не должны давать админские действия. Если войти менеджером, он не должен создавать, менять или удалять.

SQL-важно: условия поиска нужно брать в скобки, иначе фильтр будет ломаться из-за приоритета `AND` и `OR`.

Пример логики:

```sql
where (
    p.title like %s
    or p.description like %s
    or c.title like %s
    or m.title like %s
    or s.title like %s
    or u.title like %s
)
and (%s = 0 or s.id = %s)
order by p.quantity asc
```

## 12. Обработка ошибок

Сообщения должны быть нормальными, экзаменационными и полезными.

| Ситуация | Сообщение |
|---|---|
| Пустой логин или пароль | "Введите логин и пароль." |
| Неверные данные входа | "Пользователь с указанными данными не найден." |
| Не выбран товар для редактирования | "Выберите товар для редактирования." |
| Не выбран товар для удаления | "Выберите товар для удаления." |
| Товар есть в заказе | "Товар присутствует в заказе, удаление невозможно." |
| Не выбран заказ | "Выберите заказ." |
| Ошибка сохранения | "Не удалось сохранить данные. Проверьте заполнение полей." |
