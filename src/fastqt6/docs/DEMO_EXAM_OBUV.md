## Запуск проекта

```powershell
cd <папка проекта>
.\.venv\Scripts\activate.ps1
python -m pip install -r requirements.txt
python -m src.main
```

Важно: запускать именно из корня проекта командой `python -m src.main`. Если запускать `python .\src\main.py`, Python не увидит пакет `src`.

## Самое критичное для «4»

1. Импортировать `sql/shoes.sql` без ошибок.
2. Запустить проект командой `python -m src.main`.
3. Проверить входы: гость, клиент, менеджер, администратор.
4. Убедиться, что клиент и гость не видят поиск, фильтр, сортировку, заказы и CRUD.
5. Убедиться, что менеджер видит поиск, фильтр, сортировку и заказы, но не может добавлять/редактировать/удалять.
6. Убедиться, что администратор может делать CRUD товаров и заказов.
7. Показать эксперту ER-диаграмму, блок-схему и SQL-скрипт.

## Что добивает до «5»

1. Не должно быть аварийных падений при пустых полях, неверном логине, повторном выборе уже существующего изображения.
2. После добавления, редактирования и удаления список должен обновляться сразу.
3. Поиск, фильтр и сортировка должны работать совместно, без кнопки «Найти».
4. В карточке товара должны быть все поля из ТЗ.
5. У скидочного товара цена должна быть зачеркнута, новая цена показана рядом.
6. У товара с `quantity = 0` должна быть отдельная подсветка.
7. Заказ, редактирование заказа и удаление заказа должны быть видны только в логике админа.
8. Репозиторий должен быть чистым: без `.venv`, `.idea`, `__pycache__`.
9. В `docs` должны лежать ER-диаграмма, блок-схема и чек-лист скриншотов.

## Частые ошибки

PowerShell:

```powershell
.\.venv\Scripts\activate.ps1
deactivate
```

Команды `source ...` в PowerShell не работают, это команда из bash/Linux.

Правильный запуск модуля:

```powershell
python -m src.main
```

SQL для поиска, фильтра и сортировки:

```sql
where (
    p.article like %s
    or p.title like %s
    or p.description like %s
    or c.title like %s
    or m.title like %s
    or s.title like %s
    or u.title like %s
)
and s.title = %s
order by p.quantity asc
```

Скобки вокруг `or` обязательны. Без них фильтр по поставщику ломается из-за приоритета `AND` и `OR`.
